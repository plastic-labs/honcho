Get Honcho running locally and make your first API calls in under 10 minutes.

## Prerequisites

Before you begin, make sure you have:

- **Python 3.8+** installed on your system
- **Docker & Docker Compose** for running dependencies
- **Git** for cloning the repository

## Step 1: Clone the Repository

```bash
git clone https://github.com/plastic-labs/honcho.git
cd honcho
```

## Step 2: Set Up Environment

### Install Dependencies

```bash
# Install UV package manager (recommended)
pip install uv

# Install project dependencies
uv sync

# Activate virtual environment
source .venv/bin/activate  # On macOS/Linux
# or
.venv\Scripts\activate     # On Windows
```

### Environment Configuration

Create a `.env` file in the project root:

```bash
cp .env.example .env
```

Edit the `.env` file with your configuration:

```env
# Database
DATABASE_URL=postgresql://testuser:testpwd@localhost:5432/honcho

# OpenAI (for embeddings)
OPENAI_API_KEY=your_openai_api_key_here

# Authentication (optional)
USE_AUTH=false

# Sentry (optional)
SENTRY_DSN=your_sentry_dsn_here
```

## Step 3: Start Services

### Start Database

```bash
# Start PostgreSQL with pgvector extension
docker-compose up -d database
```

### Run Database Migrations

```bash
# Apply database migrations
uv run alembic upgrade head
```

### Start the API Server

```bash
# Start Honcho API server
uv run fastapi run src/main.py
```

The API will be available at `http://localhost:8000`

### Start Background Workers (Optional)

```bash
# In a separate terminal, start the deriver workers
uv run python -m src.deriver
```

## Step 4: Make Your First API Call

### Create a Workspace

```bash
curl -X POST "http://localhost:8000/v1/workspaces" \
  -H "Content-Type: application/json" \
  -d '{"id": "my-first-workspace"}'
```

### Create a Peer (User)

```bash
curl -X POST "http://localhost:8000/v1/workspaces/my-first-workspace/peers" \
  -H "Content-Type: application/json" \
  -d '{"id": "user-123"}'
```

### Create a Session

```bash
curl -X POST "http://localhost:8000/v1/workspaces/my-first-workspace/sessions" \
  -H "Content-Type: application/json" \
  -d '{
    "id": "session-123",
    "peers": {
      "user-123": {}
    }
  }'
```

### Add Messages

```bash
curl -X POST "http://localhost:8000/v1/workspaces/my-first-workspace/sessions/session-123/messages" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      {
        "content": "Hello, this is my first message!",
        "peer_id": "user-123"
      }
    ]
  }'
```

### Query with Dialectic API

```bash
curl -X POST "http://localhost:8000/v1/workspaces/my-first-workspace/peers/user-123/chat" \
  -H "Content-Type: application/json" \
  -d '{
    "queries": "What can you tell me about this user?",
    "session_id": "session-123",
    "stream": false
  }'
```

## Step 5: Explore the API

### API Documentation

Visit `http://localhost:8000/docs` to explore the interactive API documentation.

### Key Endpoints

- **Workspaces**: `/v1/workspaces`
- **Peers**: `/v1/workspaces/{workspace}/peers`
- **Sessions**: `/v1/workspaces/{workspace}/sessions`
- **Messages**: `/v1/workspaces/{workspace}/sessions/{session}/messages`
- **Chat**: `/v1/workspaces/{workspace}/peers/{peer}/chat`

## Troubleshooting

### Common Issues

**Database Connection Failed**
- Ensure PostgreSQL is running: `docker-compose ps`
- Check your DATABASE_URL in `.env`

**OpenAI API Errors**
- Verify your OPENAI_API_KEY is set correctly
- Ensure you have sufficient OpenAI credits

**Port Already in Use**
- Change the port: `uv run fastapi run src/main.py --port 8001`
